# Genomix 多智能体实验设计与预期结果

本文档补充 `src/experiments` 模块的代码概览，逐个实验说明研究意图、配置对比、核心指标以及在成功运行后的期望输出。研究人员可以据此核对生成的表格与图表是否与方案一致。

## 通用设置
- **数据集输入**：通过 `--dataset` 传入 `.h5ad` 文件路径；工具链会在首次运行时生成工作目录，后续实验复用缓存以缩短时间。
- **任务集合**：`task_library.build_task_suite()` 提供 14 个用户意图，覆盖细胞注释、差异分析、通路富集、细胞通讯、记忆回溯、知识问答与模糊澄清等使用场景。
- **运行次数**：`--runs-per-task` 控制每个任务/配置组合的重复次数，默认 1，可按实验设计提升统计置信度。
- **输出目录**：`ExperimentSuite` 在指定目录生成原始运行记录、按配置/任务聚合的 CSV、六张图（Fig 3A–3F）以及意图分类混淆矩阵数据。

## 实验 1：多智能体 vs ReAct 基线
- **目的**：验证规划器、记忆和检索协同带来的效率、准确性与解释性提升。
- **配置**：
  - *multi_agent*：全部能力开启（规划、重规划、记忆、RAG、知识问答）。
  - *react_baseline*：单智能体 ReAct 模式，禁用重规划、记忆与检索，由模型直接序列化调用工具。
- **指标**：任务成功率、平均工具调用次数、耗时、关键字召回率。
- **预期输出**：`summary_by_config.csv` 中 multi_agent 成功率应高于 baseline；`fig_3a_grouped_bars.png` 展示成功率/时长对比。

## 实验 2：规划器消融
- **目的**：评估禁用规划步骤对复合任务的影响。
- **配置**：
  - *multi_agent*：默认运行。
  - *no_planner*：`planner_mode="disabled"`，同时关闭记忆与检索。
- **指标**：复合任务成功率、工具调用次数、重规划触发次数。
- **预期输出**：`fig_3b_planner_violin.png` 显示无规划配置的工具调用呈现更大方差，成功率显著下降。

## 实验 3：重规划与容错
- **目的**：在 30% 工具调用概率注入故障，比较带/不带重规划器的恢复能力与开销。
- **配置**：
  - *replanner*：启用重规划并允许最多 4 次尝试。
  - *no_replanner*：禁用重规划，仅记录失败。
- **指标**：故障恢复率、平均尝试次数、额外耗时。
- **预期输出**：`failure_recovery_table.csv`（由 `failure_recovery_table` 生成）在 `replanner` 条目中应看到更高的恢复概率；`fig_3c_survival.png` 的生存曲线应证明多次尝试逐渐收敛。

## 实验 4：记忆组件贡献
- **目的**：衡量会话记忆在跨轮对话与上下文保持中的作用。
- **任务**：`memory_task_sequences()` 返回配对任务（播报患者 ID 与随后追问）。
- **配置**：
  - *memory_enabled*：启用记忆，连续使用相同 `thread_id`。
  - *memory_disabled*：禁用记忆，使用冷启动线程。
- **指标**：记忆准确率、响应中是否提及既往信息、耗时差异。
- **预期输出**：`memory_scores.csv`（由 `memory_scores` 导出）和 `fig_3d_memory_radar.png` 中记忆开启配置在准确率和满意度维度明显占优。

## 实验 5：知识检索消融
- **目的**：评估 RAG 与 `dataset_bio_qa` 对回答事实性问题的贡献。
- **配置**：
  - *rag_enabled*：默认配置，允许调用知识检索。
  - *rag_disabled*：禁用 RAG 与数据集问答工具。
- **指标**：事实准确率（BLEU/F1 近似，基于关键词命中）与响应延迟。
- **预期输出**：`fig_3e_accuracy_latency.png` 中启用检索的点应位于更高准确率区域，延迟略增或相近；`knowledge_accuracy.csv` 反映同样趋势。

## 实验 6：意图识别准确率
- **目的**：在 200 条标注提示上验证意图分类器的性能，以及其与下游成功率的关联。
- **流程**：禁用工具执行，仅运行 `run_objective` 的意图识别路径，收集预测标签。
- **指标**：F1 分数、混淆矩阵、主要误分类类型。
- **预期输出**：`intent_predictions.csv` 与 `intent_classification_report.csv` 提供详细指标，`fig_3f_confusion.png` 展示混淆矩阵热图。

## 运行结果校验建议
1. 检查 `metadata.json` 是否记录正确的运行次数、随机种子和数据集路径。
2. 若 `aggregated_results.csv` 中某配置的 `tool_calls` 极高，确认缓存目录是否可写；首次运行后重试应显著加速。
3. `failure_report.json`（若存在）归纳了工具失败详情，可用于定位异常。
4. 图表为空或缺失时，确认相应实验的 DataFrame 非空；CLI 在检测到空数据时会跳过绘图并写入日志。

通过以上说明，研究人员可以快速比对实验输出是否满足论文撰写所需的定量与可视化证据。
